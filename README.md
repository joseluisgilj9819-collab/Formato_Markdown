#Markdown: Historia, utilidad y casos de uso

## 1) ¿Qué es Markdown?
Markdown es un lenguaje de marcadoq ue permite aplicar formatos de texto de manera rapida y sencilla.

## 2) Origen e historia de Markdown
Markdown se creo en 2004 por John Gruber y Aaron Swartz con el proposito de crear una herramienta de escritura que eliminara las fricciones entre pesamientos del autor y el formato digial. La idea principal no fue crear un nuevo lenguaje de programacion.

## 3) ¿Para qué sirve Markdown?
Markdown sirve como un traductor universal que convierte texto simple en contenido con diseño profesional.
Tiene distintas utilidades, entre las que se encuentran:

1: Documentacion de Software: Es el uso mas importante. Si entras a GitHub o cualquier plataforma de programacion, los archivos README.md, los cuales explican como funciona un proyecto, estan escritos en Markdown. Tambien permite incluir fragmentos de codigo que se ven limpios y organizados.

2: Toma de notas tectica y personal: Herramientas como Obsidian, Logseq o Notion utilizan Markdown. Sirve para crear "cerebros digitales" donde puedes conectar ideas rapidamente sin pelearte con el formato. Al ser texto plano, puedes buscar palanras dentro de miles de notas de forma instantanea.

3: Eficiencia: Sirve para ahorrar tiempo. Una vez que aprendes que ** significa negrita y # significa titulo, dejas de depender de menus visuales y tu velocidad al crear documentos aumenta drasticamente. Ademas, te aseguras de que tus documentos se podran leer en cualquier ordenador del futuro.

## 4) Casos de uso reales
Aqui te dejo 4 ejemplos de uso real donde utilizas Markdown.
1: Documentacion tecnica y GitHub: Cuando descargas un programa o visitas un proyecto de codigo abierto, la carpeta contiene un archivo llamado README.md.

2:Aplicaciones de Cerebro digital: Markdown es el motor de las aplicaciones de notas mas populares actualmene como Obsidian, Logseq o Notion. El cual en lugar de usar menus pesados, usas # para titulos o [[enlaces]] para conectar notas entre si, lo que permite crear bases de conocimiento personales que son ultrarapidas de navegar.

3: Chats de trabajo (Slack, Discord, Microsoft Teams): Cuando quieres resaltar algo en un mensaje de chat, muchas veces utilizas Markdown. Unos ejemplos serian:
Si escribes *palabra* en Slack, se vuelve negrita, Si usas el acento grave `codigo`, creas una caja de texto especial para resaltar comandos o terminos tecnicos.

4: Plataformas de contenido y blogs (Ghost y Reddit): Muchos sistemas modernos de publicacion han abandonado los editores visuales lentos por Markdown, por ejemplo, en reddit, el "Modo Markdown" es el preferido por los usuarios avanzados para dar formato a sus comentarios, y en Ghost o Static Site Generators, los escritores redactan sus articulos completos en archivos Markdown porque es mucho mas rapido de subir y pprocesar que un archivo de word.

## 5) Ventajas y desventajas

### Ventajas
Simplicidad y enfoque: Solo necesitas aprender 4 o 5 simbolos basicos para empezar de forma profesional.
                       Puedes dar formato a todo el documento sin despegar los dedos del teclado.
                       
Longevidad y portabilidad: Se escribe en un archivo .txt, lo que quiere decir que es un texto plano.
                           Pruedes empezar a escribir una nota en tu telefono movil, edtarla en un Mac y acabar de hacerla en un ordenador con Linux sin preocuparte de que te salten errores de compatibilidad.
                           
Versatibilidad tecnica: Se puede transformar facilmente a casi cualquier formato: HTML(paginas web), PDF, ePub(libros electronicos) o PPT(presentaciones).
                        Para programadores es ideal porque permite ver exactamente que palabra se cambio en un documento usando herramientas como Git.
                        
Peso y rendimiento: Un documento de 10 paginas en Markdown ocupa apenas unos pocos Kilobytes, mientras que otros procesadores de texto podrian pesar Megabytes.
                    Cualquier aplicacion que use Markdown carga de orma instantanea los archivos, sin importar si tienes cioentos o miles de notas.

### Desventajas
Falta de estandar unico: El problema es que un documento que se ve perfecto en GitHub puede no verse igual en Notion o en Obsidian

No es "Lo que ves, es lo que tienes": Aunque escribas un codigo, siempre se vera como un texto plano.
                                      Para ver el diseño fianl, necesitas un editor que tenga Modo de previsualizacion o Vista de lectura.

Control de diseño muy limitado: No puedes cambiar facilmente el color de una palabra en especifico.
                                No puedes elegir la fuente (tipo de letra) dentro del documento.
                                No puedes alinear un parrafo a la derecha o justificarlo de forma nativa.
                                Si necesitas un diseño muy especifico y milimetrico para una revista o folleto, Markdown se queda muy corto.

Gestion compleja de archivos multimedia: Insertar una imagen en Markdown no es tan simple como arrastrar y soltar.
                                         Markdown solo guarda una ruta o enlace de la imagen.
                                         Si mueves la imagen de carpeta o le cambias el nombre en tu ordenador, el enlace se rompe y la imagen desaparece del documento.

## Que son los cerebros digitales
Un cerebro digital es un sistema externo de gestion del conocimiento. Basicamente , es una extension de tu memoria biologica creada mediante herramienta digitales para almacenar, organizar y conectar ideas de forma que nunca lo olvides.
